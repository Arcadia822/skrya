# Agent-Native 多话题信息采集分析系统设计

## 项目定义

这是一个 `agent-native` 的多 topic 信息采集与分析系统。

它不是传统资讯产品，也不是后台驱动的内容平台。它的控制面是文件系统，执行面是 agent + skill，采集层尽量复用现成能力，系统自己的核心价值放在“判断什么值得看”上。

第一阶段落地的话题是 `k-entertainment`，但整体架构从一开始就支持多 topic 扩展。每个 topic 都是独立目录，拥有自己的需求描述、信源配置和分析标准。所有 topic 操作都必须显式指定 `topic-id`。

## 这个系统解决什么问题

用户通常不是缺信源，而是缺下面四种能力：

1. 把模糊需求整理成可持续追踪的目标。
2. 把分散内容整理成事件。
3. 把大量候选内容压缩成每天值得看的少量结果。
4. 在必要时对单个事件继续做更深判断。

所以它更像一个个人情报工作区，而不是一个简单阅读器。

## 核心原则

1. 配置全部文件化，不做后台配置界面。
2. 维护通过 agent + skill 完成，不依赖人工逐项编辑。
3. topic 是隔离单元，所有 topic 操作都必须显式指定 `topic-id`。
4. 通用流程尽量统一，topic 差异主要体现在判断标准上。
5. 正常输出默认不展示 source list，不暴露内部调试信息。
